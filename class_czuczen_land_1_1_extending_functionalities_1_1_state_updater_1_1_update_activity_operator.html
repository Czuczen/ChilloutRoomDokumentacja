Detailed Description

Operator do aktualizacji stanu gry gracza. Nie może być async bo entityhistory nie zdąży utworzyć zmiany. Musi być synchroniczny. ISingletonDependency dla wiekszej wydajności. Task.Run dla wiekszej wydajności. Bez tych rozwiązań sianie rośliny trwało około 12 sekund dlatego, że każdy event wykonywał się po kolei. Dzięki Task.Run event przychodzi, wrzuca działanie do wątku i zwalnia kolejkę dla kolejnego eventu. Działania i tak wykonują się wręcz równocześnie. Z końcówkami ing (EntityUpdatingEventData) nie działa. Problem z jednostką pracy w PlantationController. Zamyka objectcontext czyli potrzebny using ale i tak nic nie pomagał bo przerywało transakcje.
